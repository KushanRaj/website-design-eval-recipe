Synthetic website replication tasks: diversity, quality, and agent behavior

This document shows the task set as an environment design artifact: generated multi-page websites, screenshot-only agent inputs, hidden oracle manifests, and browser-state grading. The focus here is not the reward internals; it is whether the generated tasks are varied, realistic, and useful for observing how coding agents reproduce website designs.

Core claim The task set is diverse not only by topic, but by page grammar: dashboards, catalogs, booking flows, financial calculators, sports standings, scientific profiles, editorial music pages, maps, filters, tabs, accordions, and selected states.
The agent sees only `/app/reference/screenshots/*.png`. Oracle source, replay manifest, metric config, and evaluator internals stay hidden under verifier-only task assets.

1. Task atlas: generated, not crawled

Each task is a synthetic oracle website generated from scratch, then frozen into screenshots and hidden replay metadata. This avoids crawling existing websites while still creating realistic browsing surfaces.

3. What the agent generally does

Strengths

  • Usually creates the expected page count and route scaffold.
  • Often captures global art direction: palette, mood, nav, hero/footer.
  • Builds recognizable components: cards, grids, menus, dashboards, filters, forms.
  • Can reproduce strong static views when the page grammar is obvious.

Failure modes

  • Card density, spacing, and row/column geometry drift.
  • Stateful views are missed or only cosmetically implied.
  • Rendered text/content structure changes even when the page looks plausible.
  • Custom diagrams, maps, rosters, brackets, and inventory walls are approximated.
Why this is useful for RL The task set produces a spectrum rather than a binary outcome. Good candidates preserve structure and states; medium candidates reproduce the vibe but miss local details; weak candidates look plausible while failing layout/content fidelity.

11. What this demonstrates

Task quality

  • Every task has at least 5 pages, and most have 10+ screenshots.
  • State coverage makes tasks richer than static page matching.
  • The hidden verifier boundary is clean: screenshot-only for the agent, oracle assets only for the verifier.
  • The set covers institutional, editorial, commerce, nonprofit, media, sports, science, and operational UI.

Agent behavior

  • Strong at broad visual identity and page scaffolding.
  • Weaker at exact density, CSS geometry, custom diagrams, and repeated component consistency.
  • Weakest at alternate states and specific interaction captures.
  • The reward spectrum broadly matches human intuition: better reproductions score higher, plausible rewrites score lower.
Final read This task set is a credible first dataset for the work trial: diverse generated websites, clean Harbor packaging, screenshot-only agent inputs, hidden replay evidence, and a reward surface that exposes meaningful reproduction failures.
